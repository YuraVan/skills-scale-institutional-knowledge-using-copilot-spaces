# OctoAcme — Project Management Docs Index

This folder contains standardized guides for the key stages of project management at OctoAcme. These processes help teams align on expectations, deliver reliably, and continuously improve project outcomes.

## Project Management Process Overview
- Customer-first, iterative delivery with clear roles, measurable criteria, and a focus on frequent feedback.
- Structured stages: Initiation, Planning, Execution, Release, Risk Management, Retrospective.
- Playbooks include one-pager templates, backlogs, checklists, risk registers, escalation paths, and reporting examples to support consistent delivery.

## Process Doc Index
- [Project Management Overview](octoacme-project-management-overview.md) — How OctoAcme runs projects
- [Project Initiation Guide](octoacme-project-initiation.md) — Steps to validate and launch a new initiative
- [Project Planning](octoacme-project-planning.md) — Backlog, estimation, Definition of Done, and roadmap
- [Execution & Tracking](octoacme-execution-and-tracking.md) — Daily standups, workflows, QA, and demos
- [Risk Management & Communication](octoacme-risks-and-communication.md) — Risk table, stakeholder comms, and escalation paths
- [Release & Deployment Guide](octoacme-release-and-deployment.md) — Release cycles, deployments, and incident response
- [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md) — Running retros and tracking improvements
- [Roles and Personas](octoacme-roles-and-personas.md) — Role summaries for product, project, and engineering leads

## How to use these docs
- Start with the Project Management Overview to understand the end-to-end flow.
- Use the Initiation and Planning playbooks when proposing new work.
- Follow Execution & Tracking and Release guides during delivery, and capture learnings in Retrospective docs.
- Keep roles & personas visible to clarify responsibilities across the team.
